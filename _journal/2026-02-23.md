## Session 2 ‚Äî Chat Routing + JMAP Send Reliability

### Chat page TypeError

`/chat` ‚Üí legacy `ChatController` passed `conversation` + `templates`, but `chat.tsx` expected `session` + `availableModels` ‚Üí `Object.entries(undefined)` ‚Üí TypeError.

**Fix:** Rerouted `/chat` to `Agent\ChatController`. Legacy SSE chat moved to `/sse-chat`. Added types (`AgentSession`, `AvailableModels`, stream events) to `types/agent.ts`. Fixed case mismatch (`'Chat'` ‚Üí `'chat'`), double AppLayout wrapping, `sidebarSessions` ‚Üí `sidebarConversations`, broadcast channels.

### JMAP sendEmail invalidResultReference

`jmap-jam` `requestMany` with `emailId: '#draft'` fails ‚Äî creation ID back-references don't work in the library. Raw curl works fine.

**Fix:** Split into two calls: `Email/set` (get real ID) ‚Üí `EmailSubmission/set` (use actual ID).

### VITE_REVERB_* build-time vars

WebSocket connected to `wss://localhost:8081` ‚Äî local `.env` baked into JS. Must build on server where `.env` has production values. Installed bun on mko.
