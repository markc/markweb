# 2026-02-28

## Session 1: ZFS ARC Tuning + Local LLM Setup

### What changed

Reduced ZFS ARC from ~40GB to 16GB to free RAM for local LLM inference. Cleaned up stale ollama models and pulled current-generation coding models.

### ZFS ARC reduction

EliteMini has 96GB DDR5, but ZFS ARC was consuming ~40GB (`c_max` = 43GB). For NVMe-only workloads, ARC provides diminishing returns — NVMe random reads are ~50us vs ARC's ~1us, but the 40x difference matters less than having RAM available for LLM inference.

Reduced to 16GB:
- Immediate: `echo 17179869184 > /sys/module/zfs/parameters/zfs_arc_max`
- Persistent: `/etc/modprobe.d/zfs.conf` → `options zfs zfs_arc_max=17179869184`

Available memory went from ~28GB to ~54GB.

### Ollama model cleanup

### Performance observations

`qwen3-coder-next` at 51GB on CPU: ~5 minutes for a PHP function, 90% CPU usage, machine unusable during inference. Quality was good but interactive use impractical. `qwen2.5-coder:32b` at 19GB is the practical sweet spot for this hardware.
