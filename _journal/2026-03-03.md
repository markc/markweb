### Fixed sync-from-meshd clobbering timestamps

The `mesh:sync-from-meshd` schedule (every minute) was overwriting
`last_heartbeat_at` with stale `last_seen_secs` values from meshd, making nodes
appear stuck at old timestamps even when AMP heartbeats were arriving fine.
Fixed to only set `last_heartbeat_at` on bootstrap (no existing value).

### Quieted dev output

Dropped Pail from `composer dev` (not useful at 1s frequency), added `--quiet`
to `artisan serve` and `schedule:work`. The `[server]` access logs and
`[scheduler]` execution output were the noise, not Pail.

Commit: `8c6d65d`

### gcwg socket permissions

Root cause of gcwg being stuck at "Xm ago": meshd socket was `root:root 0660`,
Laravel scheduler runs as `u1002` and couldn't connect. The `isAvailable()`
check (`file_exists()`) passed but curl failed with "Could not connect to
server". The silent `catch (\Throwable)` swallowed the error — temporarily
added warning logging to diagnose, then reverted.

Fix: `chmod 666 /run/meshd/meshd.sock` + `ExecStartPost` in meshd.service
on gcwg to persist across restarts.

## Session 3: gcwg container isolation

### LXCFS loadavg virtualisation

`/proc/loadavg` inside gcwg was showing the host's load, not the container's.
LXCFS was already installed (Incus dependency) with bind-mounts active, but
the `--enable-loadavg` flag was missing. Added a systemd override:

```
/etc/systemd/system/lxcfs.service.d/override.conf
[Service]
ExecStart=
ExecStart=/usr/bin/lxcfs --enable-loadavg --enable-pidfd /var/lib/lxcfs
```

After `systemctl restart lxcfs && incus restart gcwg`, the container reports
its own load (near zero when idle) instead of the host's.

### ZFS quota and df cleanup

All ZFS datasets were showing the full 1.3T pool size. Set refquotas:

| Dataset | Quota | Used |
|---------|-------|------|
| `containers/gcwg` (root) | 20G | 5.3G |
| `gc-postgres` | 5G | 67M |
| `gc-stalwart-blobs` | 5G | 2.8M |
| `gc-stalwart-data` | 2G | 6.9M |
| `gc-valkey` | 1G | 1M |

Root quota set via `incus config device override gcwg root size=20GB`, the
rest via `zfs set refquota=`.

Removed the SSL certs bind-mount (`incus config device remove gcwg certs`)
which was leaking the host root filesystem into `df`. Certs were already
copied into the container anyway.

Added `alias df='df -x tmpfs -x devtmpfs -x efivarfs -x devpts'` to
`/etc/bash.bashrc` inside gcwg to hide virtual filesystem noise.

### Gotchas discovered

- `schedule:work` caches the schedule file at startup — must restart the process after deploying changes to `routes/console.php`
- `ExecStartPost` in systemd is needed to chmod the meshd socket after creation — `RuntimeDirectory` only controls the directory, not the socket
- `file_exists()` returns true for unix sockets even when the process can't connect (permission denied) — `isAvailable()` gives a false positive
- meshd backs off callback retries after failures — if Laravel is briefly down during `composer dev` restart, meshd stops trying and needs a restart itself
