# 2026-03-04

## Session: Ollama provider + spam cascade design

### Ollama as shared LLM provider

Implemented Ollama as a first-class chat provider for all 4 agent interfaces. Created `OllamaChatService` with sync/streaming chat, model discovery, multi-instance support (local cachyos + pve3). Integrated into `AgentRuntime` — when provider='ollama', bypasses laravel/ai and calls Ollama directly.

Key discovery: code models (qwen2.5-coder) produce garbled output when given the full agent system prompt (~6 KB of AGENTS.md + SOUL.md + TOOLS.md + MEMORY.md + security delimiters + pgvector memories). Fixed by adding `buildMinimal()` to ContextAssembler — code models get conversation history only with a one-line system prompt. Detection is automatic via `OllamaChatService::isCodeModel()`.

Added multi-instance support with host picker in TUI — cachyos has code models + phi4, pve3 has qwen3.5 chat models.

### Deep dive: system prompt architecture

Traced and documented the full context assembly pipeline — SystemPromptBuilder loading markdown files, ContextAssembler appending semantic memories from pgvector (hybrid vector + keyword search with RRF), security delimiters for prompt injection defence. Wrote explanatory email to Charles at Bion explaining memories vs embeddings in non-technical terms.

### Three-tier spam cascade design

Designed a tiered spam classification architecture combining spamlite (Bayesian, Rust) with pgvector semantic search and LLM judgment:

1. spamlite Bayesian — 90% of messages, microseconds, free
2. pgvector k-NN — 7% escalated, 150ms, free/local
3. Haiku/Gemini Flash — 3% escalated, 500ms, ~$2/year

Each tier catches what the previous one can't: known patterns, semantic similarity, then intent/manipulation. Built into spamlite via Cargo feature flag (`semantic`) — zero-dep binary by default, HTTP calls to Ollama + markweb API when enabled. Directly relevant to email agent interface security (ai@motd.com).
