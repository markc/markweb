markweb is a multi-agent mesh platform built by Mark Constable. Every node in the mesh runs its own AI agent, mail server, database, DNS server, and embedding model, and they all communicate with each other using plain-text Markdown over WebSocket channels carried across an encrypted WireGuard network. The whole thing is built on Laravel 12, Inertia 2, and React 19.

The project exists because managing personal infrastructure across multiple servers meant too many tabs, too many logins, and too many single-purpose tools that did not talk to each other. markweb consolidates everything into one application that runs identically on every node. One app, one login, one place to see everything.

markweb is the control plane for what we call a NetServa v4.0 mesh. Each node in the mesh runs an identical service stack: FrankenPHP for web serving and reverse proxy, Stalwart Mail Server for IMAP, JMAP, SMTP, and spam filtering, PostgreSQL with pgvector for relational storage and vector embeddings and full-text search, PowerDNS for authoritative DNS and mesh service discovery, Ollama with nomic-embed-text for local embedding generation, WireGuard for encrypted inter-node transport, Laravel Reverb for real-time WebSocket streaming and mesh events, and SabreDAV for CalDAV and CardDAV calendar and contact sync. Every node is self-contained. It can serve web traffic, send and receive mail, resolve DNS, and run AI inference completely independently of the other nodes.

The critical design choice is Markdown as the wire protocol. All inter-node communication and all agent-to-user conversation flows as plain-text Markdown over WebSocket channels. Every message in the system is simultaneously human-readable, natively understood by large language models, and machine-parseable. There is no serialisation layer, no binary framing, no impedance mismatch between what the agent thinks and what the network carries.

Each node runs its own AI agent with its own memory, tools, and local context. An agent on web.motd.com knows that node's mail, DNS, and system state. An agent on web.kanary.org knows its own. Because they share a common Markdown protocol over WireGuard, agents can coordinate across nodes. One agent can ask another to check a DNS record, verify mail delivery, or report system health, and the response is the same Markdown a human operator would read. No API translation, no schema negotiation. The mesh becomes a distributed team of specialist agents, each grounded in its own infrastructure, communicating in a format that is natively understood by every LLM. This is the difference between AI that controls one server and AI that operates a network.

On top of the node service stack, markweb provides three application layers. The first is the AI Agent Platform, which offers multi-model chat with tool use, sandboxed code execution, semantic memory, and scheduled routines. It supports seven LLM providers including Anthropic, OpenAI, Gemini, Groq, xAI, DeepSeek, and OpenRouter, with Anthropic Claude as the default. The agent runtime is built on laravel/ai with real-time WebSocket streaming via Laravel Reverb. The second layer is JMAP Webmail and PIM, a full email client backed by Stalwart Mail Server over JMAP, with CalDAV and CardDAV contacts and calendars via SabreDAV. The third is Mesh Command and Control, a real-time dashboard for the WireGuard cluster where each node heartbeats every thirty seconds to the primary, syncs full mesh state, and broadcasts updates live via Reverb.

The backend is organised into service domains. The Agent service handles the core AI runtime. AgentRuntime orchestrates message handling across synchronous and streaming flows. IntentRouter classifies messages as commands, queries, or tasks and dispatches slash commands. SessionResolver manages agent sessions with appropriate trust levels. ContextAssembler builds LLM context by injecting system prompts and relevant semantic memories from pgvector. ModelRegistry returns available models filtered by which providers have configured API keys. The Tools service handles agent tool execution with four built-in tools: BashTool for shell commands, SandboxedBashTool for isolated Proxmox container execution, HttpRequestTool for web requests, and CurrentDateTimeTool. The Mail service integrates with Stalwart over JMAP using the jmap-jam client library. The Memory service generates embeddings via Ollama and stores them in pgvector for semantic search across sessions. The Security service scans agent inputs and outputs for prompt injection attempts. The Sandbox service manages a Proxmox container pool for isolated code execution. The Routines service handles scheduled and recurring agent actions. The Dav service provides SabreDAV backends for CalDAV, CardDAV, and principal management. The Mcp service exposes a Model Context Protocol server for MCP-compatible clients.

When a user sends a chat message, it hits the Agent ChatController which dispatches a ProcessChatMessage job. The job calls AgentRuntime which assembles context including system prompt and memories, resolves available tools, and selects the model. The laravel/ai package streams the response via an AnonymousAgent, broadcasting tokens in real-time over Reverb. Session lifecycle events broadcast on one channel while stream tokens broadcast on another, so the frontend can update both the conversation list and the active chat simultaneously.

The frontend uses React 19 with TypeScript delivered via Inertia 2 with no client-side routing. Components are built on Radix UI primitives. Styling uses Tailwind CSS 4 with five OKLCH colour schemes called Crimson, Stone, Ocean, Forest, and Sunset, each with light and dark modes. The interface uses a Dual Carousel Sidebar layout with two independently navigable sliding panels flanking a central workspace. The left sidebar has four panels for navigation, conversations, documentation, and mailboxes. The right sidebar has three panels for theme selection, token usage statistics, and notifications. Wayfinder generates type-safe TypeScript route helpers and server action types. The Streamdown component renders streaming markdown in real-time as the agent responds.

The current mesh has three nodes. The primary production node is mko at web.kanary.org on WireGuard address 172.16.1.210. The second production node is mmc at web.motd.com on WireGuard address 172.16.1.9. The development workstation is cachyos at web.goldcoast.org. Every node's entry point follows a web.* subdomain convention, so each node has exactly one canonical URL gated by HTTP basic auth before you reach the application login page.

Mesh networking works through heartbeats, sync, and broadcasting. Each secondary node runs a systemd timer that posts to the primary every thirty seconds over WireGuard with bearer token authentication. The payload includes the node name, WireGuard IP, and optional metadata. Each node also writes its own heartbeat directly to its local database on the same schedule. Secondary nodes pull full mesh state from the primary every thirty seconds so every node has a complete picture of the cluster. Nodes without a heartbeat for ninety seconds are marked offline. Every heartbeat triggers a MeshNodeUpdated event broadcast via Reverb so the dashboard updates live without polling. Node identity is entirely config-driven through MESH_NODE_NAME and MESH_NODE_WG_IP environment variables.

The project requires PostgreSQL with pgvector, a Stalwart Mail instance for JMAP, and Ollama with nomic-embed-text for embeddings. Development uses composer dev which starts five concurrent processes: artisan serve, the Reverb WebSocket server, a queue worker, Pail log tail, and the Vite dev server. Every change follows a deploy cycle of commit, push, pull on server, rebuild, and clear caches. The frontend is always built on the server because Vite environment variables get baked into JavaScript at build time.

markweb was born on 23 February 2026 in a single intense weekend session. Three existing Laravel projects called laraclaw for AI agents, laramail for JMAP webmail, and a mesh monitoring prototype were merged into one unified platform. The concept, architecture, and every line of code were created by Mark Constable working in close collaboration with Claude Code, Anthropic's AI coding agent. What would normally be weeks of integration work was accomplished in days through continuous human-AI pair programming.
