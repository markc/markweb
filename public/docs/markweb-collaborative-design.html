The MarkWeb

A decentralized collaborative mesh where nodes communicate over WireGuard tunnels via WebSockets, speak Markdown as their lingua franca, and run local AI models to process, generate, and transform content.

↓ Full Architecture Below ↓
The Concept

What Exactly Is a "MarkWeb"?

Imagine a network where every node is a small computer (a Raspberry Pi, a laptop, a VPS) running three things: a WireGuard tunnel for encrypted connectivity, a WebSocket server/client for real-time bidirectional messaging, and a local language model (like Ollama running Phi-3, Llama 3.2, or Mistral) that reads, writes, and transforms Markdown.

Markdown becomes the universal data format — not just for display, but as a structured communication protocol. Nodes send each other Markdown documents with embedded metadata (via YAML frontmatter), and local models can parse, summarize, translate, respond to, or collaboratively edit those documents in real time.

🎯 Key Insight: Markdown is both human-readable AND machine-parseable. Unlike JSON or Protobuf, a human can read the raw wire format. Unlike HTML, it's trivially easy for an LLM to generate correctly. This makes it the perfect lingua franca for a human+AI collaborative network.

💡 Why WireGuard + WebSockets? WireGuard operates at Layer 3 (network), creating a virtual private IP network. WebSockets operate at Layer 7 (application), providing framed, bidirectional messaging. Network Topology

How the MarkWeb Mesh Works

🕸️ Full Mesh Topology: With WireGuard, every node has a direct encrypted tunnel to every other node. WebSocket connections ride inside these tunnels. Each node runs both a WS server (to accept connections) and WS clients (to connect to peers). The result: any node can send a Markdown document to any other node — or broadcast to all — with sub-millisecond encryption overhead.

Component Deep Dives

Why Each Piece Matters

🔒

WireGuard — The Tunnel

WireGuard creates a Layer 3 encrypted mesh. Every node gets a private IP (e.g., 10.0.0.x) and can reach every other node as if on a LAN. Key advantages:

  • NAT traversal — works through firewalls
  • ~3% overhead — near-zero performance cost
  • Cryptokey routing — identity = public key
  • Roaming — survives IP changes
Tools like Netmaker, Tailscale, or innernet automate WireGuard mesh configuration so you don't hand-manage every peer.
🔌

WebSockets — The Messenger

WebSockets provide persistent, full-duplex, framed messaging over TCP. Unlike HTTP request/response, either side can send at any time. Options for the MarkWeb:

  • Laravel Reverb — if your nodes run PHP/Laravel
  • Node.js ws/µWebSockets — lightweight, fast
  • Rust tokio-tungstenite — minimal resource usage
  • Python websockets — pairs well with ML tooling
Each node runs both a WS server (port 6001) and N WS clients connecting to every peer's server. This creates the logical mesh within the WireGuard network.
📝

Markdown — The Language

Markdown as wire format is unconventional but perfect for a human+AI network:

  • Human-readable — debug by reading the wire
  • LLM-native — models produce perfect MD naturally
  • Structured via YAML frontmatter — machine metadata
  • Rich content — tables, code, lists, links
  • CRDT-friendly — text-based = mergeable
YAML frontmatter acts as the "header" (routing, typing, versioning) while the Markdown body is the "payload". It's like HTTP but human-readable end-to-end.
🤖

Local AI Models — The Brain

Each node runs a local model via Ollama or llama.cpp. The model can:

  • Summarize incoming documents
  • Respond to queries from other nodes
  • Transform content (translate, reformat, expand)
  • Merge conflicting edits intelligently
  • Generate new content from prompts
Different nodes can run different models. Node A might have Phi-3 for speed, Node B runs Llama 3.2 for quality. The MarkWeb leverages heterogeneous intelligence.

markweb-node/server.js — WebSocket server per nodeJAVASCRIPT
import { WebSocketServer } from 'ws';
import { parseMarkwebMessage, createMessage } from './protocol.js';
import { queryOllama } from './ai.js';

const NODE_ID = process.env.NODE_ID || 'node-a';
const WG_IP   = process.env.WG_IP   || '10.0.0.1';
const PORT    = 6001;

// Listen on WireGuard interface only — not exposed to internet
const wss = new WebSocketServer({
  host: WG_IP,   // ← binds to WireGuard IP only!
  port: PORT,
});

const peers = new Map();  // peerId → WebSocket

wss.on('connection', (ws, req) => {
  console.log(`[${NODE_ID}] Peer connected from ${req.socket.remoteAddress}`);

  ws.on('message', async (raw) => {
    const msg = parseMarkwebMessage(raw.toString());

    if (msg.frontmatter.type === 'presence') {
      peers.set(msg.frontmatter.from, ws);
      console.log(`[${NODE_ID}] Registered peer: ${msg.frontmatter.from}`);
    }

    if (msg.frontmatter.ai_directive) {
      // Forward the Markdown body to local AI model
      const aiResponse = await queryOllama(msg.frontmatter.ai_directive, msg.body);
      const reply = createMessage({
        type: 'ai.response',
        from: NODE_ID,
        to: msg.frontmatter.from,
        in_reply_to: msg.frontmatter.doc_id,
      }, aiResponse);
      ws.send(reply);
    }

    // Relay to other peers if broadcast
    if (msg.frontmatter.to?.includes('*')) {
      for (const [id, peer] of peers) {
        if (id !== msg.frontmatter.from && peer.readyState === 1) {
          peer.send(raw.toString());
        }
      }
    }
  });
});

console.log(`[${NODE_ID}] MarkWeb node listening on ${WG_IP}:${PORT}`);
markweb-node/mesh.js — Connect to all peersJAVASCRIPT
import WebSocket from 'ws';
import { createMessage } from './protocol.js';

const NODE_ID = process.env.NODE_ID;

// List of all peers in the WireGuard mesh
const PEERS = [
  { id: 'node-b', ip: '10.0.0.2' },
  { id: 'node-c', ip: '10.0.0.3' },
  { id: 'node-d', ip: '10.0.0.4' },
  { id: 'node-e', ip: '10.0.0.5' },
];

const connections = new Map();

function connectToPeer(peer) {
  const ws = new WebSocket(`ws://${peer.ip}:6001`);

  ws.on('open', () => {
    console.log(`[${NODE_ID}] Connected to ${peer.id}`);
    connections.set(peer.id, ws);

    // Announce ourselves
    ws.send(createMessage({
      type: 'presence',
      from: NODE_ID,
      model: 'phi-3',
      capabilities: ['summarize', 'translate', 'code-review'],
    }, `# ${NODE_ID} is online\n\nRunning **Phi-3** and ready to collaborate.`));
  });

  ws.on('message', (data) => {
    handleIncoming(peer.id, data.toString());
  });

  ws.on('close', () => {
    console.log(`[${NODE_ID}] Lost connection to ${peer.id}, reconnecting...`);
    connections.delete(peer.id);
    setTimeout(() => connectToPeer(peer), 5000);
  });

  ws.on('error', () => {}); // reconnect handles it
}

// Connect to all peers
PEERS.forEach(connectToPeer);

// Broadcast to entire mesh
export function broadcast(markdown) {
  for (const [id, ws] of connections) {
    if (ws.readyState === WebSocket.OPEN) ws.send(markdown);
  }
}
markweb-node/protocol.js — Parse & create MarkWeb messagesJAVASCRIPT
import yaml from 'js-yaml';

/**
 * Parse a MarkWeb message:
 *   ---
 *   type: document.update
 *   from: node-a
 *   ---
 *   # Hello World
 *   This is the body.
 */
export function parseMarkwebMessage(raw) {
  const match = raw.match(/^---\n([\s\S]*?)\n---\n([\s\S]*)$/);
  if (!match) {
    return { frontmatter: {}, body: raw };
  }
  return {
    frontmatter: yaml.load(match[1]) || {},
    body: match[2].trim(),
    raw,
  };
}

/**
 * Create a MarkWeb message from metadata + Markdown body
 */
export function createMessage(metadata, markdownBody) {
  const fm = yaml.dump({
    ...metadata,
    timestamp: new Date().toISOString(),
  });
  return `---\n${fm}---\n\n${markdownBody}`;
}

markweb-node/ai.js — Ollama integrationJAVASCRIPT
const OLLAMA_URL = 'http://localhost:11434';
const MODEL = process.env.MARKWEB_MODEL || 'phi3:mini';

/**
 * Send a directive + Markdown to the local Ollama model.
 * Returns the AI's Markdown response.
 */
export async function queryOllama(directive, markdownBody) {
  const systemPrompt = `You are a MarkWeb node AI. You receive Markdown documents
from the mesh network and respond in Markdown. Your current directive is: ${directive}.
Always respond with valid, well-structured Markdown.`;

  const response = await fetch(`${OLLAMA_URL}/api/generate`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: MODEL,
      prompt: markdownBody,
      system: systemPrompt,
      stream: false,
    }),
  });

  const data = await response.json();
  return data.response;
}

/**
 * Example: Ask the local model to summarize a document
 */
export async function summarize(markdown) {
  return queryOllama(
    'Summarize the following document in 3 bullet points',
    markdown
  );
}

 * Example: Smart merge of conflicting edits

export async function smartMerge(docA, docB) {
  return queryOllama(

// app/Events/MarkwebMessage.php
class MarkwebMessage implements ShouldBroadcast
{
    use Dispatchable, InteractsWithSockets, SerializesModels;

    public function __construct(
        public string $markdown,     // Full MarkWeb message (frontmatter + body)
        public string $fromNode,
        public string $docId,
    ) {}

    public function broadcastOn(): Channel
    {
        // Presence channel = mesh awareness (who's online)
        return new PresenceChannel('markweb-mesh');
    }
}

// app/Services/MarkwebAI.php
class MarkwebAI
{
    public function process(string $markdown): string
    {
        $parsed = Yaml::parseFrontmatter($markdown);
        $directive = $parsed['ai_directive'] ?? 'none';

        if ($directive === 'none') return $markdown;

        // Call local Ollama
        $response = Http::post('http://localhost:11434/api/generate', [
            'model'  => config('markweb.model', 'phi3:mini'),
            'prompt' => $parsed['body'],
            'system' => "Directive: {$directive}. Respond in Markdown.",
            'stream' => false,
        ]);

        return $response->json('response');
    }
}

markweb-node/crdt.js — Y.js document sync over WebSocketJAVASCRIPT
import * as Y from 'yjs';
import { broadcast } from './mesh.js';
import { createMessage, parseMarkwebMessage } from './protocol.js';

// One Y.Doc per shared document
const docs = new Map();  // docId → Y.Doc

export function getOrCreateDoc(docId) {
  if (!docs.has(docId)) {
    const ydoc = new Y.Doc();
    const ytext = ydoc.getText('markdown');

    // When local edits happen, broadcast the update
    ydoc.on('update', (update) => {
      const encoded = Buffer.from(update).toString('base64');
      broadcast(createMessage({
        type: 'crdt.update',
        from: process.env.NODE_ID,
        doc_id: docId,
        to: ['*'],
      }, `\'\'\'crdt\n${encoded}\n\'\'\'`));
    });

    docs.set(docId, { ydoc, ytext });
  }
  return docs.get(docId);
}

// Apply remote update
export function applyRemoteUpdate(docId, base64Update) {
  const { ydoc } = getOrCreateDoc(docId);
  const update = Buffer.from(base64Update, 'base64');
  Y.applyUpdate(ydoc, update);
}

// Get current Markdown content
export function getMarkdown(docId) {
  const { ytext } = getOrCreateDoc(docId);
  return ytext.toString();
}
🔀 AI-Assisted Merge: When CRDTs produce awkward merges (e.g., two people rewrote the same paragraph differently), the local AI model can be invoked to intelligently reconcile the content — something CRDTs alone can't do. This is a MarkWeb superpower: CRDT for structure, AI for semantics.
